## Context

当前定时任务使用APScheduler的CronTrigger配置，每天固定时间执行。需要添加工作日判断，只在工作日（交易日）执行任务。

## Goals / Non-Goals

**Goals:**
- 周六、周日不发送邮件
- 法定节假日不发送邮件
- 工作日正常发送邮件

**Non-Goals:**
- 不处理调休（周末上班的情况）
- 不处理临时休市公告

## Decisions

### 决策1：使用中国法定节假日API

**选择方案：** 使用开源的中国法定节假日数据

**备选方案：**
- 硬编码节假日列表（维护成本高）
- 使用第三方API（依赖外部服务）

**理由：** 使用开源数据文件，每年更新一次，简单可靠

### 决策2：在任务执行前判断

**选择方案：** 在每个任务函数开始时判断是否工作日

**备选方案：**
- 修改APScheduler的触发器（复杂）
- 使用单独的调度服务（过度设计）

**理由：** 简单直接，改动最小

### 决策3：节假日数据来源

**选择方案：** 使用GitHub上的中国法定节假日数据项目

**数据格式：** JSON文件，包含每年的节假日和调休日期

**更新方式：** 每年初手动更新一次

## Risks / Trade-offs

- **风险：** 节假日数据不准确
  - **缓解：** 使用多个数据源交叉验证

- **风险：** 调休情况处理不当
  - **缓解：** 当前版本不处理调休，后续版本可扩展

## Migration Plan

1. 创建 `src/utils/workday.py` 工具函数
2. 修改 `src/scheduler/jobs.py`，在任务开始时调用工作日判断
3. 添加节假日数据文件 `data/holidays.json`
4. 测试工作日判断逻辑
