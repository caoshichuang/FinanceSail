# 小红书股票内容自动运营系统

## 概述

构建一个AI驱动的小红书股票内容自动运营系统，实现每日自动采集A股、港股、美股数据，AI生成分析内容，邮件推送通知，人工一键发布。

## 目标

- 每日自动发布2-4篇股票市场分析内容
- 覆盖A股、港股、美股三大市场
- 重点追踪明星股表现和热点事件
- 提供IPO新股分析内容
- 预留个股订阅功能接口（第二阶段实现）

## 用户价值

### 对小红书用户
- 每日固定时间获取市场总结（09:00美股，17:00A股港股）
- 跨市场一站式视角，节省时间
- 明星股表现追踪，持仓用户更关注
- IPO热点分析，打新用户刚需

### 对运营者（你）
- AI自动生成内容，节省创作时间
- 只需邮件确认后发布，1-2分钟完成
- 内容差异化竞争（跨市场+固定时间+明星股追踪）
- 预留变现接口（付费群、个股订阅）

## 内容规划

### 每日固定内容

| 时间 | 内容类型 | 说明 |
|------|---------|------|
| 09:00 | 昨日美股总结 | 三大指数、中概股、明星股表现 |
| 17:00 | A股+港股总结 | 大盘指数、热点板块、明星股、资金流向 |

### 不定期内容（有则发）

- **热点个股分析**：涨跌停、明星股涨跌超3%、重大事件
- **IPO内容**：新股申购提醒、热门新股分析、上市首日表现
- **重大财经事件**：政策解读、行业事件、市场异动

## 技术方案

### 技术栈

| 组件 | 选型 | 说明 |
|------|------|------|
| 服务器 | 阿里云/腾讯云轻量服务器 | 国内，2核2G |
| AI模型 | DeepSeek API | 国内访问稳定，性价比高 |
| 数据源 | AKShare + Tushare | 国内财经数据API |
| 图片生成 | Playwright + HTML模板 | 渲染封面和字卡 |
| 通知方式 | QQ邮箱SMTP | 直接包含文案+图片 |
| 部署 | Docker + GitHub Actions | 容器化 + CI/CD |

### 工作流程

```
定时触发（服务器）
    ↓
数据采集（AKShare/Tushare）
    ↓
AI生成内容（DeepSeek）
    ↓
图片渲染（Playwright）
    ↓
发送QQ邮件（文案+图片）
    ↓
人工确认发布（复制粘贴到小红书）
```

## 个股订阅功能（第二阶段）

### 功能定位
- 付费群增值服务
- 信息提醒，非投资建议
- 用户自主决策

### 可以做
- 个股涨跌幅提醒（涨跌超X%）
- 个股公告提醒
- 个股财报提醒
- 个股新闻提醒
- 个股趋势分析

### 绝对不做
- 推荐买入/卖出

- 提供买卖时机
- 任何形式的投资建议

### 技术预留
- 数据库表结构预留
- 模块接口预留
- 推送渠道预留（邮件）

## 风险

### 技术风险
- API稳定性 → 多数据源备份、重试机制
- AI服务异常 → 重试机制、异常通知

### 合规风险
- 内容合规 → 严格免责，不荐股
- AI标识 → 每篇标注AI辅助创作
- 平台规则 → 持续关注小红书政策

### 运营风险
- 账号风险 → 人工发布，控制频率
- 内容质量 → 持续优化Prompt
